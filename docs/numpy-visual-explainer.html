Part 1 — The ndarray

Every array has a shape.

A NumPy ndarray is a grid of numbers where every cell holds the same type of value. The shape tells you how that grid is organised — how many rows, how many columns. Read shape (4, 5) as: "4 rows, 5 columns."

🔤 Key Terms — Plain English
ndarray
N-dimensional array. "N" just means it can have 1, 2, 3 or more dimensions. For data science, 2D (a table) is most common.
shape
A tuple describing the grid size. (4, 5) = 4 rows, 5 columns. Always read as (rows, columns).
dtype
Data type — what kind of number is stored. int64 = whole numbers. float64 = decimals. All cells must be the same type.
ndim
Number of dimensions. A flat list = 1. A table = 2. A stack of tables = 3. Most arrays you'll use are 2D.
axis 0
The direction going down the rows. sum(axis=0) = add up each column from top to bottom.
axis 1
The direction going across the columns. sum(axis=1) = add up each row left to right.

Axis Direction Guide

When you call mean(), sum(), or std(), NumPy needs to know which direction to collapse. This is the axis argument.

axis=0 ↓ (down the rows)
Collapses rows — gives one result per column.
arr.mean(axis=0) → average of each region across all categories.
axis=1 → (across the columns)
Collapses columns — gives one result per row.
arr.mean(axis=1) → average sales per category across all regions.
Memory tip: axis=0 goes in the direction of the first number in shape (rows). axis=1 goes in the direction of the second number (columns).
Part 2 — Indexing & Slicing

Pick exactly what you need.

NumPy uses arr[rows, cols] to select data. Slicing uses the start:stop syntax — just like Python lists but in two dimensions simultaneously. Remember: the stop index is excluded — 0:3 gives rows 0, 1, 2 (not 3).

How to read arr[row, col]

arr[2, 3]
→ Row 2, Column 3. One single value.

arr[0:2, :]
→ Rows 0 and 1, all columns. : alone means "everything".

arr[:, 1:4]
→ All rows, columns 1, 2, 3 (stop=4 excluded).

arr[1, :]
→ All columns from row 1 only. Returns a 1D array (one row flattened).

⚠️ Common mistake: arr[1] selects all of row 1. arr[1, 1] selects one specific cell. Don't confuse them.
Part 3 — Boolean Masking

Filter by condition, not by position.

A Boolean mask is a True/False array created by a comparison like arr > 100. When you use a mask to index an array, NumPy returns only the cells where the mask is True. This is how you answer questions like: "Which regions had sales above target?"

Combining Conditions

You can combine two conditions with & (AND) or | (OR). Each condition needs its own parentheses — without them, Python's operator precedence gives wrong results.

# AND — both must be True
mask = (totals > 400) & (totals < 600)

# OR — either can be True
mask = (totals < 300) | (totals > 600)

# NOT — flip True to False
mask = ~(totals > 400)
Important: A boolean mask must have the same length as the dimension you are indexing. Trying to apply a mask of length 3 to an array of length 5 will raise an error.
Part 4 — Views vs Copies

A slice is a window, not a photograph.

When you slice a NumPy array, you get a view — a window into the same memory as the original. Changing the view changes the original. If you need an independent copy, you must call .copy() explicitly. This is one of the most common sources of accidental data corruption in NumPy.
